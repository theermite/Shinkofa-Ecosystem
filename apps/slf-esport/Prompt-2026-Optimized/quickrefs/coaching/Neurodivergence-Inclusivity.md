# QuickRef: Neurodivergence & Inclusivité

> Design universel pour TDAH, HPI, autisme, dyslexie et tous profils.

---

## Principes Fondamentaux

### Neurodiversité = Variation Naturelle
- Pas de "normal" vs "anormal"
- Différentes façons de penser/percevoir
- Chaque cerveau a forces et défis

### Design Universel
- Ce qui aide les neurodivergents aide TOUT LE MONDE
- Accessibilité ≠ Option, c'est la base
- Flexibilité > One-size-fits-all

---

## TDAH (Trouble Déficit Attention/Hyperactivité)

### Caractéristiques
```
Forces :
✅ Hyperfocus sur sujets passionnants
✅ Créativité, pensée divergente
✅ Énergie, enthousiasme
✅ Résolution problèmes innovante

Défis :
⚠️ Attention fluctuante
⚠️ Procrastination
⚠️ Gestion du temps
⚠️ Mémoire de travail
```

### Design pour TDAH
```
✅ Tâches courtes, découpées
✅ Récompenses immédiates visibles
✅ Gamification modérée
✅ Timers visuels
✅ Mode focus (masquer distractions)
✅ Notifications configurables (pas intrusives)
✅ Quick capture (noter idées rapidement)

❌ Longs formulaires
❌ Texte dense sans structure
❌ Deadlines lointaines sans étapes
❌ Animations distrayantes
```

## HPI (Haut Potentiel Intellectuel)

### Caractéristiques
```
Forces :
✅ Apprentissage rapide
✅ Pensée complexe, nuancée
✅ Curiosité insatiable
✅ Vision systémique

Défis :
⚠️ Impatience avec la lenteur
⚠️ Perfectionnisme paralysant
⚠️ Ennui si pas stimulé
⚠️ Hypersensibilité
```

### Design pour HPI
```
✅ Explications nuancées (pas oversimplifiées)
✅ Profondeur disponible (détails optionnels)
✅ Personnalisation avancée
✅ Défis intellectuels
✅ Connexions entre concepts
✅ Mode expert accessible

❌ Condescendance ("c'est simple")
❌ Répétitions inutiles
❌ Limitations artificielles
❌ Contenu superficiel
```

## Autisme (TSA)

### Caractéristiques
```
Forces :
✅ Attention aux détails
✅ Logique, systématisation
✅ Honnêteté, authenticité
✅ Expertise profonde (intérêts spéciaux)

Défis :
⚠️ Surcharge sensorielle
⚠️ Communication implicite
⚠️ Changements inattendus
⚠️ Contexte social
```

### Design pour TSA
```
✅ Instructions explicites (pas d'implicite)
✅ Structure prévisible
✅ Avertissement des changements
✅ Mode sombre, réduction stimuli
✅ Communication littérale, claire
✅ Routines supportées

❌ Langage ambigu, métaphores
❌ Changements d'interface surprises
❌ Surcharge visuelle/sonore
❌ Pression sociale, comparaisons
```

## Dyslexie

### Caractéristiques
```
Forces :
✅ Pensée visuelle/spatiale
✅ Résolution créative
✅ Vue d'ensemble
✅ Narration

Défis :
⚠️ Lecture/écriture
⚠️ Séquençage
⚠️ Mémoire verbale
```

### Design pour Dyslexie
```
✅ Polices lisibles (OpenDyslexic, Lexie Readable)
✅ Espacement augmenté
✅ Contrastes suffisants
✅ Support audio/vidéo
✅ Correcteur orthographique
✅ Texte structuré (bullets, headers)

❌ Blocs de texte denses
❌ Polices fantaisie
❌ Justification texte (aligné gauche mieux)
❌ Captchas textuels
```

## Checklist Accessibilité Cognitive

### Interface
- [ ] Navigation cohérente et prévisible
- [ ] Hiérarchie visuelle claire
- [ ] Mode focus disponible
- [ ] Personnalisation taille/police/contraste
- [ ] Mode sombre

### Contenu
- [ ] Langage clair et explicite
- [ ] Instructions étape par étape
- [ ] Alternatives texte/audio/vidéo
- [ ] Résumés disponibles
- [ ] Pas de jargon non expliqué

### Interaction
- [ ] Feedback immédiat et clair
- [ ] Possibilité d'annuler/revenir
- [ ] Pas de time-outs courts
- [ ] Sauvegarde automatique
- [ ] Confirmation avant actions destructives

### Notifications
- [ ] Configurables (fréquence, type)
- [ ] Non intrusives par défaut
- [ ] Mode "ne pas déranger"
- [ ] Récapitulatifs plutôt que temps réel
